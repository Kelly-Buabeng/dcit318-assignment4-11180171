Medical Appointment Booking System / Pharmacy Management System

Overview

This repository contains two .NET Core Console Applications developed using Visual Studio Code (VS Code) on macOS:

  • Medical Appointment Booking System: Manages doctors, patients, and appointments with a simple console interface.
  • Pharmacy Management System: Manages medicines and sales, including stored procedures for advanced operations, with a console interface.

The applications connect to a SQL Server database running in a Docker container, using ADO.NET for database interactions. The code is designed for macOS but can be ported to a Windows Forms application on a Windows environment if a GUI is required.

Prerequisites

  • Operating System: macOS (tested with Docker and VS Code).
  • Software:
    • Visual Studio Code with the following extensions:
      • C# Dev Kit (for C# development and debugging).
      • SQL Server (mssql) (for database management).
    • .NET SDK (version 6.0 or later recommended).
    • Docker Desktop (for SQL Server).
  • Hardware: Sufficient memory and storage for Docker (at least 4GB RAM recommended).

Setup Instructions

1. Set Up SQL Server in Docker

Pull and run the SQL Server container:

docker pull mcr.microsoft.com/mssql/server:2019-latest
docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=YourStrong@Passw0rd" -p 1433:1433 --name sqlserver -d mcr.microsoft.com/mssql/server:2019-latest

Replace YourStrong@Passw0rd with a strong password (8+ characters, including uppercase, lowercase, numbers, and symbols).

Verify the container is running:

docker ps

Check logs if needed:

docker logs sqlserver

2. Configure VS Code

Install the required VS Code extensions (C# Dev Kit, SQL Server (mssql)).

Open the project folder in VS Code:

cd MedicalBookingSystem  # or PharmacySystem
code .

3. Create the Database

Open setup.sql in the project folder.

Use the MSSQL extension to connect to SQL Server:

  • Command Palette (Cmd+Shift+P) → SQL Server: Manage Connection Profiles → Create.
  • Server: localhost,1433
  • Authentication: SQL Login
  • User name: sa
  • Password: YourStrong@Passw0rd
  • Profile Name: e.g., “Local SQL Server”

Run the setup.sql script:

  • Right-click in setup.sql → Run Query.

This creates the database (MedicalDB or PharmacyDB) and tables/stored procedures.

Verify tables and data:

  • In the SQL Server panel, expand Databases → MedicalDB or PharmacyDB → Tables.
  • Run SELECT * FROM Doctors; or SELECT * FROM Medicines;.

4. Install Dependencies

In the terminal, install NuGet packages:

cd MedicalBookingSystem
dotnet add package System.Data.SqlClient
dotnet add package Microsoft.Extensions.Configuration
dotnet add package Microsoft.Extensions.Configuration.Json
cd ../PharmacySystem
dotnet add package System.Data.SqlClient
dotnet add package Microsoft.Extensions.Configuration
dotnet add package Microsoft.Extensions.Configuration.Json

5. Update Connection String

Edit appsettings.json in each project:

  • MedicalBookingSystem/appsettings.json: 
    {
  "ConnectionStrings": {
    "MedicalDBConnection": "Server=localhost,1433;Database=MedicalDB;User Id=sa;Password=YourStrong@Passw0rd;"
  }
}
  • PharmacySystem/appsettings.json: 
    {
  "ConnectionStrings": {
    "PharmacyDBConnection": "Server=localhost,1433;Database=PharmacyDB;User Id=sa;Password=YourStrong@Passw0rd;"
  }
}

Ensure the password matches your Docker setup.

6. Build and Run

Build the project:

cd MedicalBookingSystem
dotnet build
cd ../PharmacySystem
dotnet build

Run the application:

  • MedicalBookingSystem: 
    cd MedicalBookingSystem
dotnet run
    • Options:
      • 1: List Doctors
      • 2: Book Appointment
      • 3: List Appointments
      • 4: Exit
  • PharmacySystem: 
    cd PharmacySystem
dotnet run
    • Options:
      • 1: List Medicines
      • 2: Add Medicine
      • 3: Search Medicine
      • 4: Update Stock
      • 5: Record Sale
      • 6: Exit

Example usage:

  • Book an appointment: Enter 1 for DoctorID, 1 for PatientID, 2025-08-30 10:00 for date, and a note.
  • Add a medicine: Enter Paracetamol, Pain Relief, 7.99, 200.

7. Debug (Optional)

Set breakpoints in Program.cs (click in the gutter).

Press F5 or Run → Start Debugging (select .NET Core Launch).

Features

  • Medical Appointment Booking System:
    • List available doctors.
    • Book appointments with validation (e.g., doctor availability).
    • List existing appointments.
  • Pharmacy Management System:
    • List all medicines.
    • Add new medicines.
    • Search medicines by name or category.
    • Update medicine stock.
    • Record sales and adjust stock levels.

Transitioning to Windows Forms

Why: The original design specifies a Windows Forms GUI (e.g., DataGridView, Buttons), which isn’t supported on macOS or VS Code.

Steps:

  • Set up a Windows environment (e.g., Parallels, VirtualBox, or a Windows machine).
  • Install Visual Studio Community (Windows).
  • Create a Windows Forms App (.NET Framework) project.
  • Copy the ADO.NET logic from Program.cs to form event handlers (e.g., btnBook_Click).
  • Use the Visual Studio designer to add controls (see earlier instructions).
  • Update App.config with the connection string (use host.docker.internal,1433 from a Windows VM).

Cross-Platform GUI Alternative

Avalonia (for macOS GUI in VS Code):

  • Install templates: 
    dotnet new -i Avalonia.Templates
  • Create project: 
    dotnet new avalonia.mvvm -n MedicalBookingSystemUI
  • Port console logic to Avalonia’s MVVM pattern, using controls like DataGrid.

Troubleshooting

  • Docker Issues:
    • Ensure the container is running (docker ps).
    • Check logs: docker logs sqlserver.
    • Restart if needed: docker stop sqlserver followed by docker start sqlserver.
  • Connection Errors:
    • Verify appsettings.json matches your password and port (1433).
    • Test connection in VS Code’s SQL Server panel.
  • Build Errors:
    • Run dotnet restore to install missing packages.
    • Check .NET SDK version: dotnet --version.
